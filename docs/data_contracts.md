# Data and Artifact Contracts

Raw data is ignored by default; only small, deterministic, licensed fixtures may be tracked. Record external dataset provenance and licensing. Deployment manifests use repository-relative paths for input samples, dense/sparse weights, sparse metadata, biases, expected integer outputs, and program/data images. Generated artifacts and checkpoints are ignored unless intentionally documented golden fixtures. Evaluation summaries may be committed.

Phase 1 generates ignored `data/processed/sensor_fixture/examples.jsonl` and `metadata.json`. Each example has stable `sample_id`, 16 ordered numerical features (`feature_00` through `feature_15`), integer and string labels, and one split. Fixed class order is `normal`, `inner`, `outer`, `ball`; names are synthetic vibration-fault-style placeholders. The fixture uses seed `20260623`, 128 samples per class, and deterministic per-class split membership: 360 train, 76 validation, and 76 test. It is synthetic, not a licensed real dataset. Standardization fits mean and standard deviation only on the train split, substitutes one for zero standard deviations, and applies the saved values unchanged to validation and test.

Phase 2 writes ignored `artifacts/phase2_int8/quantized_model.json`. Format `sparrowml_int8_linear_v1` carries repository-relative checkpoint identity, fixed feature/class and lane order, input scale/zero point, per-output-channel weight scales, INT8 weights, INT32 biases, preprocessing version, calibration metadata, and accumulator type. It rejects absolute checkpoint paths, invalid dimensions, non-positive scales, and values outside signed INT8/INT32 ranges.

Phase 3 writes ignored `artifacts/phase3_sparse/sparse_quantized_model.json`. Format `sparrowml_sparse_2of4_linear_v1` represents `[output_channel][input_feature]` groups of four with two retained ascending lanes, legal metadata `000` through `101`, two signed INT8 compressed weights, LSB-first packed metadata, independent sparse per-channel scales, and INT32 biases. Traversal is output-channel-major then ascending group. The artifact rejects reserved metadata, nonzero padding, absolute source paths, malformed masks, and inconsistent compressed shapes.

Phase 4 writes ignored `artifacts/phase4_export/{dense,sparse}` packages. They contain canonical `sparrowml_ir_v1`, a four-byte-aligned logical map, little-endian model image, bounded quantized fixture input, reference-generated expected outputs, symbolic commands, and SHA-256 evidence. The package is target-facing data only; it is not a Sparrow-V execution result.

Phase 5 writes ignored `artifacts/phase5_runtime/` workspaces and canonical results using `sparrowml_sparrowv_runtime_result_v1`. The adapter emits Sparrow-V's documented `sparrowv_external_sensor_workload_v1` JSON only, preserving output-major inputs and dense or sparse group ordering. Results keep portable package and checkout identities, exact expected/observed INT32 accumulators, predictions, counter availability, and semantic validation. Host elapsed time and absolute workspace paths remain diagnostic-only and are excluded from semantic determinism hashes.

Phase 6 writes ignored `artifacts/phase6_multilayer/`. `quantized_model.json` has separate input/hidden scales and two named per-channel INT8/INT32 linear layers. Its export contains `sparrowml_ir_v2_multilayer`, a four-byte-aligned layout, binary tensors, bounded quantized inputs, intermediate integer traces, expected outputs, and symbolic commands. All identities are repository-relative; the hidden activation buffer stores signed INT8 values constrained to `[0,127]`.
